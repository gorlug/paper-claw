Findings

01AGENTS.md / CLAUDE.md👍 Solid
CLAUDE.md covers commands, architecture, the library sidecar layout, and even calls out the papwer-claw module typo so agents stop "fixing" it. Slim but useful. Missing: an explicit end-to-end test path and a "what not to do" section.
02Strict types / compiler🩹 Patchy
.golangci.yml enables only gocritic, goimports, gosec, revive. No staticcheck, errcheck, govet, unused, errorlint — the typical Go strict floor is missing.
Fix: add staticcheck, errcheck, govet, unused, unparam, errorlint to the linters.enable list in .golangci.yml.
03Strict linter / formatter🩹 Patchy
gofmt is enforced via lefthook.yml, but golangci-lint is not on PATH in a fresh shell — make lint fails with command not found. The Makefile points to $(GOPATH)/bin/golangci-lint, which only exists after make setup and may not be on the user's PATH.
Fix: install via brew install golangci-lint or have make lint bootstrap it if missing (and document the version in Makefile).
04Schema validation at boundaries👍 Solid
Anthropic tool-use input schema in internal/document/classify.go enforces the document-type enum; internal/document/schema.json + schema_test.go validates the on-disk metadata.json shape.
05Logic separated from I/O👍 Solid
Classifier interface in internal/document/classify.go is injected at the boundary; document.go / metadata.go are pure and tested without network. ocr.go isolates the ocrmypdf shell-out.
06One-command bring-up👍 Solid
make setup installs golangci-lint + lefthook; make check = format + lint + test. Documented in both README.md and CLAUDE.md. Falls short of 🚀 because make setup assumes ~/.local/bin exists and is on PATH.
07Pre-commit feedback loop🩹 Patchy
lefthook.yml defines format/lint/test steps, but .git/hooks/pre-commit does not exist — lefthook install has never been run on this clone. No secret scanning (gitleaks/trufflehog) configured. CLAUDE.md claims hooks are enforced; they aren't.
Fix: run lefthook install (have make setup do this), and add a gitleaks step to lefthook.yml.
08Dead-code guardrail💀 Broken
No staticcheck, no unused, no deadcode wired up. .golangci.yml's enabled list does not include any unused-symbol detector. Dead code can sit forever.
Fix: add staticcheck and unused to .golangci.yml; optionally run go run golang.org/x/tools/cmd/deadcode in CI.
09Logs reachable from terminal🚀 Ship-Ready
CLI prints everything to stdout/stderr. cmd/paperclaw/main.go emits JSON when stdout is not a TTY — perfect for piping into jq or an agent's shell.
10Docs ↔ code drift guard💀 Broken
Significant docs exist (README.md, docs/plan.md, skills/paperclaw/SKILL.md) but nothing flags drift. If you rename a subcommand or change a flag, only humans will notice.
Fix: add a lefthook step that warns when cmd/ or internal/document/ change without a matching README.md/SKILL.md touch; or snapshot paperclaw -help output and diff it in CI.
11Agent self-test E2E👍 Solid
It's a CLI — agent can run paperclaw process/list/show/search against testdata/ PDFs and read JSON back. SKILL.md documents the path. cmd/paperclaw/main_test.go is ~15KB of end-to-end-ish coverage. Missing: an explicit "smoke-test your change" script.
12Agentic review panel💀 Broken
.claude/commands/ contains only paperclaw.md (symlink to the user-facing skill). No /review command, no REVIEW.md, no parallel-specialist setup, no tiered review.
Fix: add .claude/commands/review.md that spawns parallel reviewers (Go-idiom, security, best-practices, maintainability), and add a REVIEW.md with the ignore-list (no theoretical risks, no untouched code).
13Friction proportional to blast radius🩹 Patchy
Recent commit a650637 "delete processed files from inbox" makes process destructive on the user's filesystem. No CODEOWNERS, no dangerzone hook around the deletion path or the classifier prompts (which shape every doc's metadata forever), no named bypass env var.
Fix: wrap inbox deletion behind a PAPERCLAW_DELETE_OK=1-style bypass with a default dry-run; print a one-line "this will permanently remove N files — re-run with --confirm" before deleting.
14Tooling tuned for the agent🩹 Patchy
The format hook in lefthook.yml prints "gofmt reformatted files. Review changes and re-stage." — actionable. The lint and test hooks just exit non-zero with raw tool output. No .gitleaksignore (no gitleaks at all). No "run make format" hint on lint failure.
Fix: wrap each lefthook command with a tiny shell that appends a remediation hint on failure (e.g. ... || { echo "↳ run: make format"; exit 1; }).

  • _quarantine/ directory + processing_error.json with error & retry_hint fields — failed docs aren't silently dropped; the agent can read why something failed and what to try next.
